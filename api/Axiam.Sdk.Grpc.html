Namespace Axiam.Sdk.Grpc

Classes

AuthInterceptor

Sync-safe authorization/x-tenant-id metadata injection on every outgoing gRPC call (CONTRACT.md §5), plus an UNAUTHENTICATED single-flight refresh-and-retry (§9, D-10) that drives the EXACT SAME shared RefreshGuard the REST transport uses — never a second guard instance.

AxiamGrpcAuthzClient

gRPC authorization transport (CONTRACT.md §1/§2/§5/§9, D-10). Wraps ONE long-lived GrpcChannel (via AxiamGrpcChannel — strict TLS, §6) intercepted by an AuthInterceptor that shares the EXACT SAME RefreshGuard/session AxiamClient's REST transport uses — never a second guard instance.

AxiamGrpcAuthzClient.AccessCheck

A single authorization check request item for BatchCheckAsync(IEnumerable<AccessCheck>, CancellationToken).

AxiamGrpcChannel

Constructs the SDK's ONE long-lived gRPC channel (D-10, CONTRACT.md §6) by reusing the EXACT SAME strict-TLS HttpClientHandler configuration the REST transport uses (CreatePrimaryHandler(byte[]?, byte[]?, byte[]?)) — additive customCa chain-trust honored, never a TLS-certificate-validation bypass. The channel is disposed by whichever component constructs it (Grpc/AxiamGrpcAuthzClient.cs), mirroring the Java sibling's AutoCloseable GrpcAuthzClient and Go's ClientConn ownership model.
