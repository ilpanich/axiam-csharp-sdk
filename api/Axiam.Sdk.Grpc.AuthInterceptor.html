Remarks

The token accessor is a caller-supplied, non-blocking Func<TResult> (mirrors the Java AuthClientInterceptor's Supplier<String> and Go interceptor.go's TokenFunc) — this class NEVER calls RefreshIfNeededAsync(CancellationToken) synchronously on the hot AsyncUnaryCall<TResponse> interception path; the accessor is read once per call to build outgoing metadata, and the guard is only ever awaited (fully asynchronously) after an UNAUTHENTICATED response has already been observed.

Sync-safety: every code path in this class is async/await all the way down — there is no .Result/.Wait()/.GetAwaiter().GetResult() anywhere, so this interceptor can never sync-over-async deadlock a caller's synchronization context (there is none here to deadlock on, by construction).

Parameters

tokenAccessor Func<string>

A non-blocking accessor for the currently cached access token (null when none is available yet). MUST NOT acquire RefreshGuard's lock or perform I/O — this runs on the hot RPC path (mirrors the REST AxiamHttpMessageHandler and the Java/Go sibling interceptors' token-accessor discipline).

tenantId string

The client's configured tenant identifier (CONTRACT.md §5), injected as x-tenant-id metadata on every call — mirrors the REST X-Tenant-Id header's use of the raw configured value.

refreshGuard RefreshGuard

The SAME RefreshGuard instance the client's REST transport uses (D-10's "one guard across REST + gRPC on one client") — never a second instance.

Injects auth/tenant metadata on every outgoing unary call (CheckAccess/ BatchCheckAccess are both unary RPCs); on UNAUTHENTICATED, drives exactly one shared-guard refresh then retries the call exactly once (§9.3 — never a loop). A second failure (including a second UNAUTHENTICATED, or a failure of the refresh itself) propagates to the caller as-is.
