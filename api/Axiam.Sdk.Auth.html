Namespace Axiam.Sdk.Auth

Classes

Jwk

A single JSON Web Key from the AXIAM organization-wide JWKS document (GET /oauth2/jwks). Shape confirmed against crates/axiam-oauth2/src/oidc.rs: kty=OKP, crv=Ed25519, x = base64url-encoded raw 32-byte Ed25519 public key, with a deterministic kid. Property names are explicitly pinned via JsonPropertyNameAttribute to the exact lowercase wire field names — never rely on a naming-policy convention.

JwksDocument

The JWKS document envelope returned by GET /oauth2/jwks: { "keys": [...] }.

JwksVerifier

Local EdDSA (Ed25519) JWKS verification (D-02, CONTRACT.md §10 local-verification fast path). .NET has no native Ed25519/EdDSA support anywhere in System.Security.Cryptography (confirmed research finding — dotnet/runtime #14741/#63174 remain unimplemented through .NET 10 GA), so this class uses BouncyCastle.Cryptography's Org.BouncyCastle.Crypto.Signers.Ed25519Signer / Org.BouncyCastle.Crypto.Parameters.Ed25519PublicKeyParameters directly — the single, well-vetted, verify-only crypto dependency D-02 permits.

LoginResult

Outcome of LoginAsync/VerifyMfaAsync (CONTRACT.md §1). MFA-required is an expected outcome represented as a flag — never thrown as an exception: callers MUST check MfaRequired before assuming a session was established. Mirrors the Java sibling's LoginResult (20-03) field shape.

RefreshGuard

Single-flight token-refresh guard (D-10, CONTRACT.md §9): guarantees exactly one in-flight refresh call at a time, regardless of how many concurrent callers observe an expired/near-expiry access token. One instance is constructed per AxiamClient and shared across the REST and gRPC transports (D-10's "one guard across REST + gRPC on one client" requirement) — never a second instance per transport.

TokenPair

An access/refresh token pair with an expiry timestamp. Both token values are wrapped in Sensitive<T> (CONTRACT.md §7, D-12) so neither can leak via ToString(), JSON serialization, or a log call.
