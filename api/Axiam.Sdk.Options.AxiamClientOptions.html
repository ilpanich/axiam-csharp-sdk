Typed, options-pattern configuration for AxiamClient (Claude's Discretion, 21-RESEARCH.md/21-CONTEXT.md D-07). BaseUrl and TenantId are required here for the future AddAxiam()/ IOptions<AxiamClientOptions> DI registration path (plan 21-06) — AxiamClient's own tenant-required constructor (this plan, SC#1) sources the tenant/base-URL from its own explicit positional parameters instead, so SC#1's compile-time guarantee never depends on whether an options object happens to be supplied at all.

ClientCertificatePem

PEM-encoded client-certificate chain presented for mutual-TLS (mTLS) client authentication (CONTRACT.md §6.1): AXIAM binds this X.509 identity — signed by the tenant's organization CA — to a service account or IoT device. Applies to both the REST and gRPC transports of the same AxiamClient. Opt-in: null (the default) leaves the SDK's bearer-cookie behavior unchanged and presents no client certificate. MUST be set together with ClientKeyPem — supplying exactly one of the two is rejected with an ArgumentException at client construction. Presenting a client certificate NEVER relaxes strict server verification (§6.1 rule 2); this is a separate code path from CustomCaPem's server-trust callback.

public byte[]? ClientCertificatePem { get; init; }

Property Value

byte[]

ClientKeyPem

PEM-encoded private key (PKCS#8 or PKCS#1) matching ClientCertificatePem, used for mutual-TLS client authentication (CONTRACT.md §6.1). Secret material (§7): it is never logged, serialized, or exposed via a public getter beyond this options record it is set on (mirrors CustomCaPem). MUST be set together with ClientCertificatePem; supplying exactly one of the two is rejected with an ArgumentException at client construction.
