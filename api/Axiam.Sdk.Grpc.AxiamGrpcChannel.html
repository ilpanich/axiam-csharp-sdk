Class AxiamGrpcChannel

Axiam.Sdk.Grpc

Constructs the SDK's ONE long-lived gRPC channel (D-10, CONTRACT.md §6) by reusing the EXACT SAME strict-TLS HttpClientHandler configuration the REST transport uses (CreatePrimaryHandler(byte[]?, byte[]?, byte[]?)) — additive customCa chain-trust honored, never a TLS-certificate-validation bypass. The channel is disposed by whichever component constructs it (Grpc/AxiamGrpcAuthzClient.cs), mirroring the Java sibling's AutoCloseable GrpcAuthzClient and Go's ClientConn ownership model.

Methods

Create(Uri, byte[]?, byte[]?, byte[]?)

Builds a single GrpcChannel targeting target over HTTP/2, configured with the same strict-TLS handler the REST transport uses (§6) — the additive customCaPem escape hatch is honored, but NO code path in this method (or in CreatePrimaryHandler(byte[]?, byte[]?, byte[]?)) installs an unconditional TLS-validation-bypass delegate. Callers are responsible for applying an AuthInterceptor via channel.Intercept(...) and disposing the returned channel when done.

public static GrpcChannel Create(Uri target, byte[]? customCaPem, byte[]? clientCertPem = null, byte[]? clientKeyPem = null)

Parameters

target Uri

The gRPC endpoint (e.g. the AXIAM server's base URL, or a dedicated gRPC target when one is configured separately from the REST base URL).

customCaPem byte[]

Optional PEM-encoded custom CA bytes (§6) — null to trust only the system trust store, exactly matching the REST transport's own default.

clientCertPem byte[]

Optional PEM-encoded client-certificate chain for mutual TLS (§6.1) — the SAME identity the REST transport presents, so a single AxiamClient authenticates consistently across both transports. MUST accompany clientKeyPem.

clientKeyPem byte[]

Optional PEM-encoded private key (PKCS#8/PKCS#1) matching clientCertPem (§6.1). MUST accompany it.
