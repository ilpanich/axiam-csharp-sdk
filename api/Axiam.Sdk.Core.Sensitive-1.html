Wraps a token-carrying value so it can never be accidentally exposed via ToString(), System.Text.Json serialization, or a public getter (CONTRACT.md §7, D-12). Only SDK-internal code may construct an instance or read the wrapped value, via Axiam.Sdk.Core.Sensitive<T>.Reveal() — there is intentionally no public accessor.

Remarks

Per CONTRACT.md §7's C# row ("Struct with ToString() override returning "[SENSITIVE]""): ToString() and the paired SensitiveJsonConverter<T> both always emit the literal "[SENSITIVE]", regardless of the wrapped value. Equals(object?), Equals(T) and GetHashCode() are overridden to make a Sensitive<T> never value-comparable: Equals(object?) always returns false and GetHashCode() always returns a constant. This is deliberate — the default ValueType equality a struct would otherwise inherit performs a structural, field-by-field value comparison (via reflection, and, for a string field, a non-constant-time string.Equals), which would open a value-equality/timing side channel on the redacted value and would transitively leak into any compiler-synthesized record equality of a type that carries a Sensitive<T> field. Overriding to a constant closes that channel.
