AxiamAmqpConsumer

AMQP event consumer: verify-before-handler HMAC gate plus the sdks/CONTRACT.md §8 ack/nack matrix, built on RabbitMQ.Client 7.x's async consumer API with automatic connection/topology recovery enabled (D-11).

StartAsync(string, string, byte[], Func<byte[], CancellationToken, Task>, ILogger?, TimeSpan?, CancellationToken) NEVER hands an unverified delivery to the caller-supplied handler — Verify(byte[], byte[]) runs first, and the handler call site is structurally unreachable until it returns true. The remaining outcomes follow the exact §8 decision matrix:

  • HMAC verification fails → BasicNackAsync(requeue: false) + a security-event log entry that never contains the received or expected HMAC value; the handler is never invoked.
  • HMAC verifies but the NEW-4 replay-protection gate (ReplayGuard) rejects the delivery — key_version below MinKeyVersion, a stale/future issued_at, or a replayed nonce — → BasicNackAsync(requeue: false) + a security-event log entry; the handler is never invoked.
  • Handler returns normally → BasicAckAsync.
  • Handler throws PoisonMessageException (poison message) → BasicNackAsync(requeue: false).
  • Handler throws any other exception (transient/retryable) → BasicNackAsync(requeue: true).
Hmac

HMAC-SHA256 verify-before-handler primitive for inbound AMQP messages (sdks/CONTRACT.md §8, D-11).

Canonicalization is wire/insertion-order preserving, NEVER alphabetically sorted. The canonical Rust signer (crates/axiam-amqp/src/messages.rs's sign_payload) signs the serde_json struct-declaration-order serialization of the message body (with hmac_signature absent). JsonObject is backed by an ordered dictionary and preserves insertion order from parsing by default — Remove(string) mutates that same ordered map in place, preserving the relative order of all remaining keys. This is the single load-bearing property of this class: do NOT introduce an alphabetized copy, OrderBy/Sort, or a POCO deserialize-then-reserialize round-trip here — any of those would silently reorder fields and break every cross-language HMAC verification.

PoisonMessageException

Poison-message sentinel a consumer-supplied handler throws to signal that the current delivery must be nacked without requeue, rather than the default nack-with-requeue applied to any other handler exception (D-11, sdks/CONTRACT.md §8). Mirrors the Go ErrDrop sentinel / Java ErrDrop exception used by the sibling SDKs.

ReplayMetadata

The key_version/nonce/issued_at fields extracted from an already HMAC-verified AMQP message body (sdks/CONTRACT.md §8 "v2 — Replay Protection", NEW-4). Extraction happens strictly AFTER Verify(byte[], byte[]) has returned true — these values are meaningless (and MUST NOT be trusted) on an unverified body.
