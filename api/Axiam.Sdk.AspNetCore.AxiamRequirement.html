Authorization requirement carrying everything AxiamPolicyHandler needs to run a single AXIAM access check (CONTRACT.md §11): the resource/action halves (legacy "resource:action" policy-string form, e.g. "documents:read", or the structured AxiamAccessAttribute form — both funnel into this same type), plus the optional Scope and the ResourceRouteParam route value name used to resolve the target resource's UUID at request time. Built by AxiamPolicyProvider.

Parameters

resource string

The resource-type half of the policy name (e.g. "documents"). Optional — when null, PolicyName is action alone.

action string

The action half of the policy name (e.g. "read"). Required, non-blank.

scope string

Optional scope for sub-resource granularity, passed through to CheckAccessAsync verbatim.

resourceRouteParam string

The name of the route value whose UUID identifies the target resource. Defaults to DefaultResourceRouteParam ("id").
