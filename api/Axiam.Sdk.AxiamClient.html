Remarks

Owns exactly ONE Axiam.Sdk.AxiamClient.RefreshGuard and ONE Axiam.Sdk.AxiamClient.JwksVerifier per client — shared by the REST auth flow here and, by a later plan, the gRPC transport (D-10's "one guard across REST + gRPC on one client" requirement). Internal accessors (Axiam.Sdk.AxiamClient.RefreshGuard, Axiam.Sdk.AxiamClient.JwksVerifier, Axiam.Sdk.AxiamClient.CurrentAccessToken, Axiam.Sdk.AxiamClient.BaseUrl, Axiam.Sdk.AxiamClient.CustomCaPem, Axiam.Sdk.AxiamClient.TransportHttpClient) expose this seam without requiring the gRPC plan (21-05) or the ASP.NET Core plan (21-06) to edit this file.

Constructors

AxiamClient(Uri, string, AxiamClientOptions?)

The ONLY construction path (SC#1) — tenantId is required and positional; there is no overload reachable from this class that permits omitting it (CONTRACT.md §5: AXIAM is multi-tenant, there is no default tenant). A blank tenantId is a runtime guard (via TenantContext) backing this compile-time guarantee.

public AxiamClient(Uri baseUrl, string tenantId, AxiamClientOptions? options = null)

The AXIAM server's base URL.

Optional tuning (custom CA, org id/slug, timeouts, JWKS cache TTL). When omitted, sane defaults are used and BaseUrl/ TenantId are populated from baseUrl/tenantId.
