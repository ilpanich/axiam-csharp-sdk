AMQP event consumer: verify-before-handler HMAC gate plus the sdks/CONTRACT.md §8 ack/nack matrix, built on RabbitMQ.Client 7.x's async consumer API with automatic connection/topology recovery enabled (D-11).

StartAsync(string, string, byte[], Func<byte[], CancellationToken, Task>, ILogger?, TimeSpan?, CancellationToken) NEVER hands an unverified delivery to the caller-supplied handler — Verify(byte[], byte[]) runs first, and the handler call site is structurally unreachable until it returns true. The remaining outcomes follow the exact §8 decision matrix:

  • HMAC verification fails → BasicNackAsync(requeue: false) + a security-event log entry that never contains the received or expected HMAC value; the handler is never invoked.
  • HMAC verifies but the NEW-4 replay-protection gate (ReplayGuard) rejects the delivery — key_version below MinKeyVersion, a stale/future issued_at, or a replayed nonce — → BasicNackAsync(requeue: false) + a security-event log entry; the handler is never invoked.
  • Handler returns normally → BasicAckAsync.
  • Handler throws PoisonMessageException (poison message) → BasicNackAsync(requeue: false).
  • Handler throws any other exception (transient/retryable) → BasicNackAsync(requeue: true).

StartAsync(string, string, byte[], Func<byte[], CancellationToken, Task>, ILogger?, TimeSpan?, CancellationToken)

Connects to amqpUri, opens a channel with automatic recovery enabled, and registers an RabbitMQ.Client.Events.AsyncEventingBasicConsumer on queue that verifies every delivery's HMAC-SHA256 signature (§8, via Verify(byte[], byte[])) BEFORE handler is invoked, then applies the §8 ack/nack matrix documented on this class.

public Task StartAsync(string amqpUri, string queue, byte[] signingKey, Func<byte[], CancellationToken, Task> handler, ILogger? logger = null, TimeSpan? replaySkew = null, CancellationToken cancellationToken = default)

Parameters

amqpUri string

The AMQP connection URI.

queue string

The queue name to consume from.

signingKey byte[]

The per-tenant AMQP HMAC signing secret (§8.1 — obtain from the AXIAM management API; never hardcode).

handler Func<byte[], CancellationToken, Task>

Invoked ONLY after HMAC verification AND the NEW-4 replay-protection gate (ReplayGuard) both succeed. Throw PoisonMessageException to drop a message without requeue; any other exception is treated as transient and requeues.

logger ILogger

Receives the §8.4 security event on HMAC verification (or NEW-4 replay-protection) failure; the event never contains the HMAC, nonce, or issued_at value. Defaults to a silent no-op logger.

replaySkew TimeSpan?

The NEW-4 freshness window applied to each delivery's issued_at field (also sets the nonce dedup store's TTL, 2×replaySkew). Defaults to DefaultSkew (±5 minutes) when null or non-positive.

cancellationToken CancellationToken

Cancellation token for connection/channel setup and for every ack/nack call issued by the registered consumer.
