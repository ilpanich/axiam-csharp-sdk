Remarks

AXIAM's RBAC engine is additive-only (allow-wins, default-deny, SEC-040; project constraint per CLAUDE.md) — this handler calls CheckAccessAsync FRESH on every single evaluation (T-21-18). It NEVER caches an authorization decision beyond the current request; a client-side cache or short-circuit could silently diverge from the server's live additive-only decision, which this SDK must never risk.

HandleRequirementAsync(AuthorizationHandlerContext, AxiamRequirement)

Resolves the caller's user_id claim and the target resource id (from route data), then calls AxiamClient.Authz.CheckAccessAsync fresh — see the type-level remarks for why no decision is ever cached. Succeeds requirement when allowed; otherwise leaves it unsatisfied so AxiamAuthorizationMiddlewareResultHandler can map the outcome to a standardized 401/403 JSON body.
