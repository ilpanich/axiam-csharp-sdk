Remarks

The wire tenant_id/subject_id fields are resolved from the CURRENT access token's claims — preferring a signature-VERIFIED resolution via JwksVerifier (D-02) when available, falling back to an unverified decode purely as an operational hint (mirrors AxiamClient.DoHttpRefreshAsync's own established pattern) when no verifier is supplied or local verification fails — NEVER from the raw configured tenant identifier string. The real server (crates/axiam-api-grpc/src/services/authorization.rs) independently cross-validates both wire fields against its own verified JWT claims and rejects PERMISSION_DENIED on any mismatch, so a wrong hint here can only ever produce a denial — never an over-grant.
